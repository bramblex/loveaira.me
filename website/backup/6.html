【瞎扯】编程语言在精不在多？谬论！

分类： 根分类 > 技术杂谈

多年以前，总是有这样那样的人到处说什么“编程语言只要学一门就好啦”或者“编程语言在精不在多”这样的话。然而局限于当年大家那点可怜的计算机基础，所以也就不能要求大家都能理解编程语言。不过现在都那么多年过去了，一代一代的拥有良好计算机基础的科班毕业生都已经入行了，就这样还能有人拿着编程语言扯事的我就没看懂了。

▶主流编程语言就那么几类

你看现在主流编程语言那么多，每年还爆出来各种所谓的“下一代”编程语言，但是问题是这些编程语言基本就在一个圈里面绕，从来都没有出圈。这个圈是指三个编程语言范式——命令式范式、面的对象范式以及函数式范式。

所以主流的编程语言绕来绕去都还在这个圈里面，这是我们主流编程语言的通性。

▶主流编程语言所谓的“新”特新

现在主流编程语言所谓的“新”特性真的一点都不新。说句难听，这些编程语言所谓的新特性，很可能都是学术界一百年前就玩剩下的。最近的 Java8 吹那么大，还专门出了一本书来讲的“新”特性，就属于那种学术界玩剩下的东西。

所以，盲目追求这些主流编程语言的所谓新特性是没有任何意义的。努力提高计算机理论基础显然更有意义，也更有价值，甚至连你追潮流都能比别人快一步。

▶精于领域而不是精于语言

上面扯了那么多学术的东西，我们现在扯回来工程。

绝大多数人（除了那些真正在搞编程语言的人）其实都并不精于语言，他们甚至都不见得真正精于语言的使用，但是他们根本就没必要精于某个编程语言。为啥？编程语言只是一个工具，你只需要用到你需要的功能，而不需要真正去掌握工具提供给你的全部功能，以及这些功能全部的意义。

那我们所需要的“精通”是精通个啥？是精通领域而不是精通语言！比如你是搞安卓的，他是搞J2E的，你们写的真的是一样的 Java 吗？

▶经验是指在某领域的经验

除了一些特定的语言几乎只用在特定的领域以外（比如 PHP ），其他通用语言都会在多个领域使用。比如 Python 可以运用在 Web 开发、科学计算、机器人等多个领域。所以你说你是写 Python ，你到底是干嘛的？

在同一个领域，解决的问题都是相似的。领域的经验往往都是可以跨语言的，就像我折腾 Web 领域比较多，所以我可以用 PHP, Python, Ruby, Node.js 以及本博客所使用的 PureScript 来造网站、造 Web 框架，而并不需要真正“精通”这些语言（你们谁那么牛逼给我精通一个 PureScript 看看）。但是你要让我拿 Python 去做科学计算，那对不起，我真的不懂……

一个语言会在不同的领域解决不同的问题，但是解决这些问题有时候往往并不是要求你对这门语言有多精通，而在于你对这个领域有多精通。

▶写在最后

本文总结起来就是：

  1. 语言都长一样，按照规范会用就行。
  2. 领域各显高低，注重积累精益求精。
